# 제품 정의

## 한 줄 정의

서울을 방문한 한국인과 외국인에게 현재 장소와 연결된 한양의 역사 이야기를 짧고 재미있게 설명하는 역사 도슨트 서비스.

## 대상 사용자

- 서울을 여행하는 한국인
- 서울을 처음 방문한 외국인
- 경복궁, 광화문, 북촌, 종로, 한양도성 같은 장소에서 역사적 맥락을 알고 싶은 사용자

## 해결할 문제

서울의 장소는 역사적 맥락이 풍부하지만, 일반 관광객은 현장에서 그 맥락을 즉시 이해하기 어렵다.

기존 검색은 다음 문제가 있다.

- 설명이 길다.
- 현장 장소와 직접 연결되지 않는다.
- 출처를 확인하기 어렵다.
- 외국인이나 비전문가에게 어렵다.
- 음성 질의처럼 짧고 모호한 질문에 약하다.

## 제품 목표

사용자가 서울의 특정 장소에서 질문하면, 시스템은 다음을 제공한다.

- 장소와 연결된 역사적 배경
- 관련 인물, 사건, 제도
- 짧은 도슨트식 설명
- 필요 시 더 깊은 해설
- 화면용 citation
- 음성용 축약 답변

## 핵심 사용 흐름

```text
사용자 위치 또는 장소 선택
-> 질문 입력 또는 음성 질문
-> 장소명/지시어 보정
-> 관련 역사 근거 검색
-> 짧은 도슨트 답변 생성
-> citation 표시
```

## MVP

1차 MVP는 음성 앱이 아니라 RAG backend다.

필수 기능:

- 장소 catalog
- place-aware query rewrite
- citation RAG
- 짧은 관광 해설 답변
- evaluation harness

제외 기능:

- 실시간 위치 기반 route recommendation
- 완성형 모바일 앱
- 다국어 TTS
- 사용자 계정
- 결제
- AR

## 성공 기준

- 장소 기반 질문에서 관련 근거를 찾는다.
- 답변이 관광객에게 짧고 이해 가능하다.
- citation이 원문 근거를 지지한다.
- 없는 근거는 만들어내지 않는다.
- query type별 성능 차이를 평가 리포트로 설명할 수 있다.
